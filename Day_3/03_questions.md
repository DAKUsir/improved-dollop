1. what is Greedy Decoding
    - Greedy decoding is a simple decoding strategy used in text generation tasks. It works by selecting the most probable token at each step of the generation process. This means that at each time step, the model chooses the token with the highest probability as the next word in the generated sequence. While greedy decoding is computationally efficient, it can lead to suboptimal results as it may get stuck in local maxima and fail to explore other possible sequences that could be more coherent or creative.
2. what is Beam Search
    - Beam search is a more sophisticated decoding strategy used in text generation tasks. It maintains a fixed number of candidate sequences (called beams) at each time step, rather than just one as in greedy decoding. The algorithm expands each beam by generating possible next tokens and keeps only the top-k most probable sequences for the next step. This allows beam search to explore multiple potential sequences and can lead to better results compared to greedy decoding. However, it is also more computationally expensive, especially as the beam width (k) increases.
3. what is Pure Sampling
    - Pure sampling is a decoding strategy where the next token is selected randomly based on the probability distribution predicted by the model. Instead of always choosing the most probable token (as in greedy decoding) or maintaining a fixed number of candidate sequences (as in beam search), pure sampling allows for more diversity in the generated text. This can lead to more creative and varied outputs, but it may also result in less coherent or relevant text if the model's predictions are not well-calibrated. Pure sampling is often used when creativity and diversity are desired in the generated text.
4. what is Top-k Sampling
    - Top-k sampling is a decoding strategy where the next token is selected from the top-k most probable tokens predicted by the model. Instead of considering all possible tokens, top-k sampling limits the selection to a smaller subset of tokens that have the highest probabilities. This allows for more diversity in the generated text while still maintaining some level of coherence, as it avoids selecting low-probability tokens that may lead to nonsensical outputs. The value of k can be adjusted to control the trade-off between diversity and coherence in the generated text.
5. what is Top-p (Nucleus) Sampling
    - Top-p sampling, also known as nucleus sampling, is a decoding strategy where the next token is selected from the smallest set of tokens whose cumulative probability exceeds a certain threshold (p). Instead of selecting from a fixed number of top tokens (as in top-k sampling), top-p sampling dynamically determines the number of tokens to consider based on their probabilities. This allows for more flexibility and can lead to more coherent and contextually relevant text generation, as it focuses on the most probable tokens while still allowing for some diversity. The value of p can be adjusted to control the balance between diversity and coherence in the generated text.
6. what is Temperature Effect
    - Temperature is a hyperparameter used in text generation that controls the randomness of the output. A higher temperature value (greater than 1) makes the model more likely to select less probable tokens, resulting in more diverse and creative outputs. Conversely, a lower temperature value (between 0 and 1) makes the model more conservative, favoring more probable tokens and producing more coherent but less diverse text. Setting the temperature to 1 means that the model will sample from the original probability distribution without any modification. Adjusting the temperature allows for fine-tuning the balance between creativity and coherence in the generated text.
7) what is top-k and top-p sampling? How do they differ and when would you use one over the other?
    - Top-k sampling and top-p (nucleus) sampling are both decoding strategies used in text generation to introduce diversity while maintaining coherence. Top-k sampling selects the next token from the top-k most probable tokens predicted by the model, while top-p sampling selects from the smallest set of tokens whose cumulative probability exceeds a certain threshold (p). The main difference is that top-k sampling limits the selection to a fixed number of tokens, whereas top-p sampling dynamically determines the number of tokens based on their probabilities. You would use top-k sampling when you want to control the number of candidate tokens directly, and top-p sampling when you want to ensure that you are selecting from a set of tokens that collectively have a high probability, allowing for more flexibility in the number of tokens considered.    
8) What is repetition penalty and how does it affect the generated text?
    - Repetition penalty is a technique used in text generation to discourage the model from generating the same token or sequence of tokens multiple times. It works by applying a penalty to the probability of tokens that have already been generated, making them less likely to be selected again. This helps to reduce redundancy and improve the diversity of the generated text. The repetition penalty can be adjusted to control how strongly it affects the generation process, with higher values leading to more aggressive penalties and thus more diverse outputs.
